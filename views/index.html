Features

Automationg Basic Git Flow

Always taking a long time writing meaningful commit messages when code comments expain it all? Forgit will add, commit and push your code to github for you.

  • Add will stage one file at a time.
  • Commit will have a message based on the comments in the code.
  • Push will send your code to github if connected to WIFI.

Offline

If you are on a bus to work, in a park, at the beach, or even on a plane working on something you just remembered. Your code can continue to create history.

  • Add still works.
  • Commits with file based comment messages.
  • Push will not work offline.
  • Notifications will continue.

Notifications

Notifications are native to your machine. Forgit will send a notificaton for many reasons including:

  • Commit events.
  • Push events.
  • Errors.
  • Internet errors with pushing.
  • No commits.

Time Intervals

Every developer is different, so giving them an option to set times for the git flow automation is important.

  • Commit times are set in minutes.
  • Push times are set in minutes.

Setting Workspaces

When a developers workflow is different based on where or what they are working on, workspaces will give you a way to set settings based on those workflows.

  • Example naming would be work, home, freelance.
  • Select inividual repos for group.
  • Set if you want notification.
  • Setting if you want commit or push to automate.

Features

Automating Basic Git Flow

Always taking a long time writing meaningful commit messages when code comments explain it all? Forgit will add, commit and push your code to Github for you.

  • Add stages one file at a time.
  • Commit has a message based on the comments in the code.
  • Push sends your code to Github if connected to the internet.

Offline

If you are on a bus to work, in a park, at the beach, or even on a plane working on something you just remembered. Your code can continue to create history.

  • Add still works.
  • Commits with file based comment messages.
  • Push does not work offline.
  • Notifications continue.

Notifications

Notifications are native to your machine. Forgit sends a notificaton for many reasons including:

  • Commit events.
  • Push events.
  • Errors.
  • Internet errors with pushing.
  • No commits.

Time Intervals

Every developer is different, so giving them an option to set times for the Git flow automation is important.

  • Commit times are set in minutes.
  • Push times are set in minutes.

Setting Workspaces

When a developers workflow is different based on where or what they are working on, workspaces give you a way to set settings based on those workflows.

  • Example naming would be work, home, freelance.
  • Select individual repos for group.
  • Set if you want notification.
  • Setting if you want commit or push to automate.
